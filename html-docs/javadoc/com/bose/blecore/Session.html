Interface Session

  • All Known Implementing Classes:
    SimulatedSession

    public interface Session

    A session represents the communication between this device and a remote BluetoothDevice. It must be opened (via the open() method) to initiate a connection. It must be closed (via the close() method) to tear down the connection. It must be disposed (via the `dispose()` method) to remove it from the BluetoothManager that created it, allowing it to be deallocated.

    Calling open() on a session begins the connection process. This is a multi-step process that ends successfully with an instantiated Device object and an open communication channel. The process is as follows:

    - open() calls BluetoothGatt.connect() for the given device.

    - If the connection was successful (the BluetoothProfile.STATE_CONNECTED status is received indicating a successful connection), we begin discovering services on the peripheral (via BluetoothGatt.discoverServices()).

    - After services are discovered (the session received a BluetoothGattCallback.onServicesDiscovered(BluetoothGatt, int) indicating success), we attempt to instantiate a matching Device type by calling BluetoothManager.instantiateDevice(SerializedGatt, BluetoothDevice).

    - If a matching Device type is found and successfully instantiated, the session was successfully opened and the SessionDelegate is notified.

    - If any of the steps above fail, the SessionDelegate is notified.

      • refreshServices

        void refreshServices()
        Refresh GATT services. Android OS Bluetooth stack caches the list of available services and their characteristics for paired devices. New services or characteristics added (or removed) by the remote Bluetooth device firmware are thus not detected, requiring either (a) to unpair/pair cycle or (b) this method call. *Experimental API*
